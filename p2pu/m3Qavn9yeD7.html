GitHub (for) Course(s)

-------------------

Why Github pages? Dirk and Carl discussing 8/19/14

  • concern that it’s more complicated / user-unfriendly than we need - requires a bit more knowledge to get started than eg Lernanta. (or Wordpress?)
    • need to learn mark-down
    • more complicated sign-up process
  • Leads to two questions:
    • are those concerns correct?
    • if so, are the rewards/benefit worth the concerns?
      • (OR - can the concerns be removed by good tech design / overlays)

Github pages: 

  • For P2PU: easier to maintain a big/complex ecosystem in github pages, 
    • for course creators: get benefit of a higher quality ecosystem and support coz easier for us to do that in Github pages vs Wordpress
    • less spam/crap courses for us to deal with: longer onboarding for community means people know what they’re doing by the time they can build the course / no more ’course cloning’, ’fire and forget’ activity
    •  multiplicity of eg Wordpress plugins/functionality become very complex / high maintenance burden, very quickly. Dirk sees Github pages as better suited to high volume of courses with varying needs
    • we carry archiving responsibility for everything that goes up under our umbrella; putting courses on a 3rd party domain shifts archive/preservation burden to that platform
    • distance ourselves from perception of P2PU as a platform
    • want people to have stronger personal identity on the web / ownership of the courses
    • We don’t do group ownership/input to direction of the course
      • Github pages has the potential to facilitate collaboration/input on where courses should go
  • Therefore

There is an opportunity to leverage the collaborative tools that GitHub provides for collaboratively creating courses.

We are not first to think about this or execute: course fork & other platforms.

Sources of inspiration:

  • GitHub like experience for writers: https://draftin.com/
  • Git for creating books: https://github.com/GitbookIO/gitbook
  • Prose.io

Questions:

  • Currently we can leverage GitHub fairly well for collaboratively creating the content for the course. But is that really the difficult part of creating an online course? What are the difficult parts and how can we address that?

Git benefits

  • see what changed
  • compare different versions
  • allows editing by anyone
  • handle conflicts
  • allows review before merge
  • keeps track of history

GitHub benefits

  • Web UX
  • Hosting
  • API
  • Integration with other services (TravisCI)

GitHub disadvantages

  • Central platform controlled by commercial entity
  • Focus on Code
  • Lacks extra tools for learning

Other notes:

  • Mixing taking a course and creating a course. A true P2PU course should be a mix of having a framework, but the people taking the course also having the ability to shape the direction and content of the course. Practical example: Collaborative creation of a course is something we can do with GitHub, but what if people taking the course we’re able to author certain parts of the course in the same way someone would collaboratively create the course?
